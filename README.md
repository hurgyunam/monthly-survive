# Monthly Survive

> **모바일 / 다인원 개척 생존 시뮬레이션**  
> 설정만 하고 직접 컨트롤은 하지 않는다. 한 달을 지켜본 뒤, 다시 설정한다.

---

## 📋 프로젝트 소개

**Monthly Survive**는 **RimWorld**(콜로니, 생존, 다인원)와 **FM2020**(직접 조작 없음, 설정 후 관찰) 스타일을 결합한 생존 시뮬레이션 프로젝트입니다.

플레이어는 **지휘관/감독** 역할을 수행하며, 캐릭터를 직접 조작하지 않고 **정책과 규칙만 설정**합니다. 콜로니원들의 행동 우선순위를 세팅한 후 → 한 달간 시뮬레이션이 돌아가는 것을 관찰 → 결과를 보고 다음 달 정책을 재설정하는 루프를 반복합니다.

---

## 🎮 핵심 컨셉

| 원칙 | 설명 |
|:---|:---|
| **설정형 플레이** | "누가 무엇을 얼마나 할지" 규칙을 정하고, 시뮬이 그에 따라 돌아감 |
| **월 단위 사이클** | 게임 시간 1달 = 1사이클. 그 안에서 자동 진행 |
| **관찰/분석** | 로그, 통계, 이벤트로 무슨 일이 있었는지 파악 |
| **모바일 친화** | 짧은 세션, 턴제적 느낌(월 단위), 터치 UI |

## 📱 화면 구성

| 경로 | 화면 | 설명 |
|:---|:---|:---|
| `/` | **설정 화면** | 이번 달 집중 활동 선택 (채집 / 건축 / 휴식) |
| `/sim` | **시뮬레이션** | 30일(실제 30초) 자동 진행, 로그 표시 |
| `/result` | **결과 화면** | 이번 달 성과 수치, 다음 달 설정으로 이동 |

## 📅 MVP 로드맵

프로젝트는 **MVP(Minimum Viable Product)** 단위로 점진적으로 개발됩니다.

| 순서 | MVP | 내용 | 상태 |
|:---:|:---|:---|:---:|
| 1 | **MVP-01** | 첫 번째 월 사이클 (설정→관찰→재설정) | 🔄 진행 중 |
| 2 | **MVP-02** | 콜로니원 2~3명, 역할/우선순위 세팅 | ⬜ 예정 |
| 3 | **MVP-03** | 자원(식량 등), 생존 실패 조건 | ⬜ 예정 |
| 4 | **MVP-04** | 월간 리포트, 다음 달 설정 연동 | ⬜ 예정 |
