# MVP-05: 월간 리포트 + 다음 달 설정 연동

> **검증 목표:** "한 달이 끝난 뒤 결과를 보고, 그 결과를 바탕으로 다음 달 설정을 이어갈 수 있는가?"  
> → 브레인스토밍의 **"월 단위 사이클: 설정 → 시뮬 돌림 → 결과 보고 → 다시 설정"** 루프 강화.

---

## 🎯 목표

- **월간 리포트**: 한 달 시뮬 종료 시 **결과 요약(리포트)**를 명확히 보여 준다. (활동별 성과, 자원 변화, 생존 여부 등)
- **다음 달 설정 연동**: 리포트를 본 뒤 **"다음 달 설정하기"**로 넘어가서, 이전 결과를 참고해 집중활동·콜로니원·자원 목표 등을 다시 설정할 수 있게 한다.

"설정 → 시뮬 → **결과 보고** → **재설정**"이 한 사이클로 굴러가도록 피드백 루프를 완성한다.

---

## 📦 범위 (In Scope)

### 1. 월간 리포트 화면/내용
- **한 달 결과 요약** 표시
  - 콜로니원별 활동·성과 (MVP-02 연동)
  - (MVP-04 완료 시) 자원 보유량 변화, 생존 성공/실패
  - (MVP-03 완료 시) 지역·재해·약탈 이벤트 요약
- **로그 또는 요약 텍스트**로 "무슨 일이 있었는지" 파악 가능
- "다음 달 설정하기" 버튼 → 설정 화면으로 이동

### 2. 다음 달 설정 연동
- 설정 화면에 **이전 달 결과** 참고 가능 (요약 노출 또는 링크)
- 이전 달과 **동일/다른 설정**을 선택할 수 있음 (집중활동, 역할, 자원 목표 등)
- "이달 시작" 시 새 월 시뮬이 이전 결과를 이어받아 시작 (상태 초기화 규칙 명확히)

### 3. 데이터/상태
- 월별 결과 저장 (리포트용)
- "현재 달" / "다음 달 설정 대기" 등 상태 전이

---

## 🚫 범위 밖 (Out of Scope)

| 제외 항목 | 이유 |
|:---|:---|
| 장기 통계/그래프 | 우선 한 달 단위 리포트만 |
| 시설·제작 결과 리포트 | MVP-06, MVP-07 이후 확장 |

---

## ✅ 완료 조건

- [ ] 한 달 종료 시 **월간 리포트**가 표시되고, 활동·자원·생존 여부 등을 확인할 수 있다.
- [ ] **"다음 달 설정하기"**로 넘어가 이전 결과를 참고해 설정을 변경·유지할 수 있다.
- [ ] 설정 → 시뮬 → 결과 보고 → 재설정이 **한 루프**로 자연스럽게 동작한다.

---
