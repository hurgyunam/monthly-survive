# MVP-03: 초반 지역 선택 — 지역에 따라 재해·약탈·자원이 다르게 존재

> **검증 목표:** "플레이어가 초반에 지역을 고르고, 지역별로 재해·약탈·자원이 다르게 존재해 게임 변수와 긴장감이 생기는가?"  
> → [03-MVP-브레인스토밍.md](../03-MVP-브레인스토밍.md)의 **게임 변수** 항목 반영.

---

## 🎯 목표

- **초반 지역 선택**: 게임(또는 새 게임) 시작 시 플레이어가 **지역을 선택**할 수 있게 한다. 지역마다 난이도·자원·리스크가 다르게 설계된다.
- **지역별 자연 재해**: 지역에 따라 **다른 종류의 자연 재해**가 발생할 수 있게 하여 변수를 준다.
- **외부 약탈 이벤트**: 랜덤하게 **외부 세력이 약탈하러 오는 이벤트**를 넣어 변수와 긴장감을 더한다.
- **지역별 자원 차이**: 지역에 따라 **존재하는 자원(종류·풍부함)** 이 다르게 설정되어, "어디에서 살아남을지" 선택이 전략적으로 의미 있게 한다.

"어디에서 살아남을지" 선택과 "예측 못 하는 사건", "지역별 자원"으로 월 단위 시뮬에 긴장감과 전략을 더한다.

---

## 📦 범위 (In Scope)

### 1. 지역 선택
- **지역 목록** 정의 (최소 2~3종: 예 — 평원, 숲, 해안)
- 게임 시작 시 또는 "새 게임" 시 **지역 선택 UI**
- 선택한 지역이 저장되고, 해당 월 시뮬에 반영

### 2. 지역별 자연 재해
- 지역마다 **발생 가능한 재해** 종류 다르게 설정 (예: 평원=가뭄, 숲=산불, 해안=폭풍)
- 월 시뮬 중 **재해 발생** 시 자원/건강 등에 영향 (규칙 정의)
- 결과 화면 또는 로그에 "이번 달 재해: ~" 표시

### 3. 외부 약탈 이벤트
- **랜덤 이벤트**: 월 중 일정 확률로 "외부 세력 약탈" 발생
- 약탈 시 **자원 감소** 또는 **피해** 규칙 정의
- 결과/로그에 이벤트 발생 여부·결과 표시

### 4. 지역별 자원 차이
- 지역마다 **존재하는 자원 종류·풍부함** 정의 (예: 숲=목재 풍부, 평원=식량 보통, 해안=특정 자원 제한)
- 시뮬/결과에서 지역에 따라 자원 획득 가능 여부·비율 반영

### 5. 데이터/상태
- 지역 정의(이름, 재해 종류, 자원 정보, 난이도 등) 데이터
- 이벤트 종류·발생 조건·효과 데이터

## 🚫 범위 밖 (Out of Scope)

---

## ✅ 완료 조건

- [ ] 플레이어가 **초반에 지역을 선택**할 수 있고, 선택이 시뮬에 반영된다.
- [ ] **지역별로 다른 자연 재해**가 발생할 수 있으며, 재해가 결과에 반영된다.
- [ ] **외부 약탈 이벤트**가 랜덤으로 발생하고, 자원/결과에 반영되어 긴장감이 느껴진다.
- [ ] **지역에 따라 자원(종류·풍부함)** 이 다르게 존재하고, 시뮬에 반영된다.
