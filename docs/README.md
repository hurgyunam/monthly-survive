# docs - 문서 구조

> **목적:** 기획용 문서, AI/개발용 컨텍스트, 참고용 상식을 역할별로 구분합니다.

---

## 폴더 역할

| 폴더 | 대상 | 용도 |
|:---|:---|:---|
| **planning** | 사람 (기획자, PM) | 기획, 브레인스토밍, 의사결정, 로드맵 관리 |
| **ai-context** | AI (개발 어시스턴트) | 개발 시 참조할 문서 안내 및 진입점 |
| **reference** | 팀원, AI | 커밋 규칙, 네이밍 등 참고용 상식·컨벤션 |

---

## planning (기획용)

- **역할:** 프로젝트 기획, MVP 설계, 의사결정 기록
- **내용:** 로드맵, MVP 목록, 브레인스토밍, 개별 MVP 상세 스펙
- **특징:** 기획 과정과 히스토리가 함께 유지됨

```
planning/
├── README.md           # 프로젝트 개요, MVP 철학
├── design/             # 게임 핵심 설계, 설계 원칙
└── mvps/               # MVP별 상세 기획
    ├── 01-MVP-로드맵.md
    ├── 02-MVP-목록.md
    ├── 03-MVP-브레인스토밍.md
    └── MVP-01/         # 개별 MVP
        ├── README.md
        ├── MVP-01-UI-spec.md
        └── MVP-01-TODO.md
```

---

## ai-context (AI/개발용)

- **역할:** AI가 개발 시 참고할 문서의 **진입점** 및 가이드
- **내용:** 참조할 문서 경로, 현재 진행 중인 MVP 안내
- **특징:** 실제 스펙은 planning에 두고, **링크만** 유지 (중복 없음)

```
ai-context/
└── README.md   # AI용: "개발 시 읽어야 할 문서" 가이드
```

---

## reference (참고용)

- **역할:** 팀·프로젝트 공통 참고 문서 (상식, 규칙, 컨벤션)
- **내용:** Conventional Commits, (추후) 브랜치 전략, 용어집 등
- **특징:** 기획/스펙이 아닌 "어떻게 일할지"에 대한 문서만 모음

```
reference/
├── README.md                  # 이 폴더 설명 및 문서 목록
└── conventional-commits.md    # 커밋 메시지 컨벤션
```

---

## 사용 방법

- **기획/회의 시:** `planning/` 폴더의 문서를 편집·갱신
- **AI 개발 요청 시:** `@docs/ai-context` 또는 `ai-context/README.md`를 참조하게 하면, AI가 해당 가이드에 따라 올바른 planning 문서를 읽음
- **커밋/규칙 참고 시:** `reference/` 폴더에서 해당 문서 확인
