# MVP-02: 콜로니원 + 역할/우선순위

> **검증 목표:** "다수의 콜로니원에 역할/우선순위를 세팅하고, 각자 다른 활동이 시뮬에 반영되는가?"

---

## 🎯 목표

플레이어가 **2~3명의 콜로니원**을 설정하고, 각자에게 **역할 또는 우선순위**를 부여한 뒤 → 한 달 시뮬레이션에서 **각자의 활동이 구분되어 진행**되는 것을 검증한다.

MVP-01의 "한 사람 + 한 활동"에서 **다인원 + 개별 설정**으로 확장한다.

---

## 📦 범위 (In Scope)

### 1. 월 초 설정 화면 확장
- **콜로니원 2~3명** 표시 (이름/아이콘)
  - 기본값: 2명 (이름은 "콜로니원 1", "콜로니원 2" 또는 설정 가능)
  - 3명으로 확장 시: "콜로니원 3" 추가
- **역할/우선순위 배정**
  - 각 콜로니원별로 **활동 1종 선택** (채집 | 건축 | 휴식)
  - 동일 활동 중복 배정 허용 (예: A=채집, B=채집)
- "이달 시작" 버튼 → 시뮬레이션 진입

### 2. 월 중 시뮬레이션 화면 확장
- **콜로니원 2~3명 각각 카드 표시** (이름 + 배정된 활동)
- 각자가 배정된 활동에 따라 "~중..." 표시
- **로그**에 콜로니원별 구분 반영
  - 예: "Day 1: 콜로니원 1 채집 중...", "Day 5: 콜로니원 2 건축 완료..."
- 30일 종료 시 결과 화면으로 전환

### 3. 월 말 결과 화면 확장
- **콜로니원별 성과 수치** 표시
  - 예: "콜로니원 1 (채집): 50", "콜로니원 2 (건축): 35"
- "다음 달 설정하기" 버튼 → 설정 화면 복귀

### 4. 데이터/상태
- 콜로니원 목록 (이름, 배정 활동) 저장
- MVP-01의 월/시뮬 로직 유지

## 🚫 범위 밖 (Out of Scope)

| 제외 항목 | 이유 |
|:---|:---|
| 콜로니원 추가/삭제 UI | 고정 2~3명으로 시작. 추후 MVP에서 확장 |
| 능력치, 특성 | MVP-03 이후 |
| 자원(식량, 나무 등) | MVP-03 |
| 생존 실패/게임오버 | MVP-03 |
| 우선순위 순서(1순위, 2순위) | 이번 MVP는 "역할 1종만" 배정. 다중 우선순위는 이후 |
| 저장/불러오기 | MVP-04 이후 |

## 📐 시뮬레이션 로직

- 각 콜로니원은 MVP-01과 동일한 수치 공식을 사용
- 활동별: 채집 30+random(0~20), 건축 25+random(0~15), 휴식 20+random(0~10)
- 로그: 콜로니원별로 독립적으로 이벤트 발생 (선택적, 최소 3~5줄 유지)
