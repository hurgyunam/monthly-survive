# MVP-01: 첫 번째 월 사이클

> **검증 목표:** "설정 → 관찰 → 재설정" 핵심 루프가 동작하는가?

---

## 🎯 목표

플레이어가 **최소한의 설정**을 하고 → **한 달 시뮬레이션**을 관찰한 뒤 → **결과를 보고 다시 설정**할 수 있는 흐름을 구현한다.

이 MVP만으로도 **"설정형 플레이"의 맛**을 느낄 수 있어야 한다.

---

## 📦 범위 (In Scope)

### 1. 월 초 설정 화면
- **이번 달 집중 활동** 1종 선택 (3개 중 1개)
  - 옵션: `채집` | `건축` | `휴식`
- "이달 시작" 버튼 → 시뮬레이션 진입

### 2. 월 중 시뮬레이션 화면
- 게임 시간 1달 = 30일 진행 (실제 플레이: 약 30~60초 또는 속도 조절 가능)
- **1명의 콜로니원** (이름/아이콘만 있어도 됨)이 선택한 활동에 따라 "작업 중" 표시
- **간단한 로그** 3~5줄: "Day 1: 채집 중...", "Day 5: 채집 완료 10..." 등
- **일시정지/속도 조절** (1x, 2x, 4x) — 선택적, 선택 시 구현
- 30일 종료 시 자동으로 결과 화면으로 전환

### 3. 월 말 결과 화면
- 이번 달 **요약 수치** 1개 표시
  - 예: "채집 성과: 50" (선택한 활동에 따라 달라짐)
- "다음 달 설정하기" 버튼 → 설정 화면으로 복귀

### 4. 데이터/상태
- 현재 "몇 월"인지 저장 (1월, 2월, 3월...)
- 이번 달 선택값 → 시뮬 결과에 반영 (채집 선택 시 채집 수치 증가 등)

## 🚫 범위 밖 (Out of Scope)

| 제외 항목 | 이유 |
|:---|:---|
| 다수의 콜로니원 | MVP-02에서 확장 |
| 자원(식량, 나무 등) | MVP-03에서 도입 |
| 생존 실패/게임오버 | MVP-03 |
| 건물, 맵, 배치 | 이후 MVP |
| 이벤트, 선택지 | 이후 MVP |
| 저장/불러오기 | MVP-04 이후 |
| 밸런싱, 보상 루프 | 검증 후 |

## 🖥️ 화면 흐름

```
[설정 화면]                    [시뮬 화면]                    [결과 화면]
┌─────────────────┐           ┌─────────────────┐           ┌─────────────────┐
│ 이번 달 집중:   │  "시작"   │ Day 15/30       │  30일 후  │ 2월 결과        │
│ ○ 채집         │ ────────► │                 │ ────────► │ 채집 성과: 50   │
│ ● 건축         │           │ [콜로니원]      │           │                 │
│ ○ 휴식         │           │ 건축 중...      │           │ [다음 달 설정]  │
│                 │           │                 │           │       │         │
│ [이달 시작]    │           │ 로그...         │           │       ▼         │
└─────────────────┘           └─────────────────┘           └────────┬────────┘
                                                                      │
                                                                      ▼
                                                           [설정 화면] (3월 설정)
```

**라우팅:** 위 3개 화면은 React Router로 각각 다른 페이지 URL을 가진다. (예: `/setup`, `/sim`, `/result`)

## 📐 시뮬레이션 로직 (최소 규칙)

| 선택 | 30일 후 결과 (예시) |
|:---|:---|
| 채집 | `채집 성과 = 30 + random(0~20)` |
| 건축 | `건축 성과 = 25 + random(0~15)` |
| 휴식 | `휴식 성과 = 20 + random(0~10)` (수치보다는 "피로 회복" 의미) |

*실제 수치/공식은 구현 시 조정. "선택에 따라 결과가 다르게 나온다"만 보이면 됨.*

---


## ✅ 완료 조건

- [ ] 설정 화면에서 활동 1종 선택 후 "이달 시작" 가능
- [ ] 시뮬 화면에서 30일(또는 추상화된 1달)이 진행됨
- [ ] 선택한 활동이 로그/표시에 반영됨
- [ ] 결과 화면에 선택에 따른 수치가 표시됨
- [ ] "다음 달 설정" 클릭 시 설정 화면으로 돌아가며 월이 증가함
- [ ] **루프 2회 이상** 돌았을 때 이전 선택과 무관하게 정상 동작
