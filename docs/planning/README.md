## 🎯 핵심 철학: MVP 우선 개발

> "완성을 목표로 하기 전, 여러 개의 작은 MVP로 쪼개서 진행한다"

### 이 방식의 장점
- **빠른 검증**: 각 MVP마다 핵심 가치를 빠르게 검증 가능
- **리스크 분산**: 큰 규모의 실패 대신 작은 실패에서 학습
- **동기 유지**: 짧은 사이클마다 "완성" 경험을 얻음
- **유연한 피벗**: 방향 전환이 필요할 때 부담 적음
