# MVP-04: 자원(식량·목재·흙 등) 도입 + 자원 채취 계획 UI

> **검증 목표:** "자원을 도입하고, 자원 채취 계획을 세울 수 있는 UI가 있으며, 자원 부족 등 생존 실패가 시뮬에 반영되는가?"  
> → [03-MVP-브레인스토밍.md](../03-MVP-브레인스토밍.md)의 **자원 리스트** 기반으로 생존 시뮬 확장.

---

## 🎯 목표

- **자원 도입**: 식량(생존 필수), 목재·흙 등 기본 자원을 시뮬 변수로 넣고, 월 단위로 소비·획득이 반영되게 한다.
- **생존 실패 조건**: 자원 부족(예: 식량 고갈) 등으로 **생존 실패**가 발생하면 시뮬 결과에 반영하고, "다음 달"로 이어지지 않거나 패널티가 있게 한다.
- **자원 채취 계획 UI**: 플레이어가 월/사이클 단위로 **어떤 자원을 얼마나 채취할지** 계획을 설정할 수 있는 UI를 제공한다.

MVP-01·02의 "설정 → 관찰 → 결과" 루프 위에 **자원**, **생존 여부**, **채취 계획**을 올려 생존 시뮬의 핵심 루프를 검증한다.

---

## 📦 범위 (In Scope)

### 1. 자원 정의·표시
- **기본 자원 종류** 도입 (최소: 식량, 목재, 흙 — 브레인스토밍 자원 리스트 중 우선 적용할 것만)
- 월 초 **보유량/목표** 표시, 월 말 **획득·소비 결과** 표시
- (선택) 자원별 설명/용도 툴팁 — 브레인스토밍 표 참고

### 2. 자원의 시뮬 반영
- **식량**: 월 중 소비(콜로니원 수 등 반영), 부족 시 생존 실패 가능
- **목재·흙 등**: 채집/활동에 따라 획득·소비 규칙 정의 (1~2주 내 구현 가능 범위로)
- (MVP-03 완료 시) 지역별 자원 차이 반영

### 3. 생존 실패 조건
- **실패 조건** 정의 (예: 식량 0 이하, 특정 자원 고갈 등)
- 실패 시 **결과 화면**에 실패 사유·표시, "다음 달" 진행 제한 또는 경고

### 4. 자원 채취 계획 UI
- 월 초(또는 사이클 시작 전) **채취 계획** 설정 UI
  - 대상 자원 종류 선택 (식량, 목재, 흙 등)
  - 목표량 또는 우선순위(1순위, 2순위 등) 설정
- 설정된 계획이 시뮬/결과에 반영 — 결과 화면에서 달성 여부 표시

### 5. 데이터/상태
- 자원 종류·수치, 소비/획득 규칙 데이터
- 생존 성공/실패 상태 저장 및 결과 연동
- 채취 계획(자원 종류, 목표/우선순위) 저장

---

## 🚫 범위 밖 (Out of Scope)

| 제외 항목 | 이유 |
|:---|:---|
| 지역 선택·지역별 자원 정의 | MVP-03에서 다룸 |
| 시설·제작·자원 조합 | MVP-06, MVP-07에서 다룸 |
| 브레인스토밍의 전 자원(석유·우라늄 등) | 우선 기본 자원만, 확장은 이후 |

---

## ✅ 완료 조건

- [ ] **식량·목재·흙** 등 최소 자원이 월 단위 시뮬에 **소비/획득**으로 반영된다.
- [ ] **생존 실패 조건**이 정의되고, 실패 시 결과 화면에 표시·다음 달 연동(제한/경고)이 동작한다.
- [ ] **자원 채취 계획 UI**로 플레이어가 채취할 자원·목표/우선순위를 설정할 수 있고, 설정이 시뮬/결과와 연동된다.
- [ ] 플레이어가 "자원이 있어야 산다"는 생존 시뮬 감을 체험할 수 있다.

## 📌 의존 MVP

- **MVP-02**: 콜로니원 + 역할/우선순위 (채집 활동 반영)
