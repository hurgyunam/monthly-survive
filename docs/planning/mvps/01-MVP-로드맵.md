# MVP 로드맵

## 📌 사용 방법

이 문서는 **MVP 구현 순서**와 **의존 관계**를 관리합니다.  
각 MVP를 진행하기 전, 여기서 다음에 할 MVP를 확인하고 상태를 갱신하세요.

---

## 🔗 의존성 다이어그램

```
[MVP-01] ──► [MVP-02] ──┬──► [MVP-03]  (초반 지역 선택)
                        ├──► [MVP-04] ──► [MVP-06] ──► [MVP-07]
                        │     (자원+채취 계획)  (시설 건설)  (시설 제작)
                        └──► [MVP-05]  (월간 리포트 + 다음 달 설정)
```

- 화살표: "완료 후 진행 가능" 관계
- 병렬로 진행 가능한 MVP는 같은 레벨에 배치

## 📅 로드맵

| 순서 | MVP ID | MVP명 | 의존 MVP | 상태 |
|:---:|:---:|:---|:---|:---:|
| 1 | MVP-01 | 첫 번째 월 사이클 | - | ✅ |

## 📝 로드맵 작성 가이드

1. **의존성 최소화**: 가능하면 독립적으로 동작하는 MVP부터
2. **핵심 가치 우선**: 사용자가 가장 빨리 "아, 이거 쓸 만하네"를 느끼는 것부터
3. **작게 유지**: 각 MVP는 1~2주 안에 완료 가능한 규모로
