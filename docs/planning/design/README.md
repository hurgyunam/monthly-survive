# Monthly Survive - 게임 핵심 설계

> **장르:** 모바일 / 다인원 개척 생존 시뮬레이션  
> **참조:** RimWorld(콜로니, 생존, 다인원) + FM2020(직접 조작 없음, 설정 후 관찰)

---

## 🎮 핵심 컨셉

### 한 줄 요약
> **"설정만 하고, 직접 컨트롤은 하지 않는다. 한 달을 지켜본 뒤, 다시 설정한다."**

플레이어는 **지휘관/감독**이지 **직접 조작자**가 아니다.  
콜로니원들의 행동 규칙, 우선순위, 정책을 세팅하고 → 시뮬레이션이 돌아가는 걸 관찰 → 결과를 보고 다음 달 정책을 재설정하는 루프.

---

## 🔄 핵심 루프

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  1. 설정    │ ──► │  2. 관찰    │ ──► │  3. 재설정   │
│  (셋팅)     │     │  (한 달)    │     │  (피드백)   │
└─────────────┘     └─────────────┘     └─────────────┘
       ▲                                         │
       └─────────────────────────────────────────┘
```

| 단계 | 내용 | 플레이어 행동 |
|:---:|:---|:---|
| **1. 설정** | 다음 달 운영 방침 세팅 | 우선순위, 역할 배정, 정책, 건물/작업 순서 등 |
| **2. 관찰** | 시뮬레이션 실행 (한 달) | 결과 로그/리플레이 확인, 이상 징후 체크 |
| **3. 재설정** | 피드백 반영 | 성공/실패 원인 분석 → 다음 달 설정 수정 |

**직접 조작 불가:**  
캐릭터를 클릭해서 "여기 가서 일해" 같은 명령은 없음. 오직 **정책/규칙**만 설정.

---

## 📐 설계 원칙

| 원칙 | 설명 |
|:---|:---|
| **설정형 플레이** | "누가 무엇을 얼마나 할지" 규칙을 정하고, 시뮬이 그에 따라 돌아감 |
| **월 단위 사이클** | 게임 시간 1달 = 1사이클. 그 안에서 자동 진행 |
| **관찰/분석** | 로그, 통계, 이벤트로 무슨 일이 있었는지 파악 |
| **모바일 친화** | 짧은 세션, 턴제적 느낌(월 단위), 터치 UI |

## 🧩 주요 시스템 (참고용)

*MVP 단위로 쪼개서 구현. 아래는 전체 그림 참고용.*

| 시스템 | 설명 | FM/RimWorld 대응 |
|:---|:---|:---|
| **콜로니원** | 여러 명의 주민, 각각 능력치/성향 | RimWorld 콜로니스트 |
| **역할/우선순위** | "A는 채집 우선, B는 건축 우선" 설정 | FM 포지셔닝/역할 |
| **자원/생존** | 음식, 건자재, 체력 등 | RimWorld |
| **건물/시설** | 건설 순서, 배치 규칙 설정 | 둘 다 |
| **이벤트** | 랜덤/시나리오 이벤트 발생 → 자동 대응 or 선택지 | FM 매치 이벤트 |
| **월간 리포트** | 한 달 요약, 통계, 성공/실패 로그 | FM 시즌 리뷰 |

## 📅 월 단위 사이클 상세

```
[월 초] 설정 화면
  ├─ 이번 달 목표 (예: 저장고 확장, 식량 100 확보)
  ├─ 역할 배정 (A: 채집, B: 건축, C: 요리...)
  ├─ 우선순위 (건축 > 채집 > 휴식)
  └─ (선택) 특수 지시

[월 중] 시뮬레이션 자동 진행
  ├─ 플레이어는 "관찰만" (속도 조절 가능)
  ├─ 이벤트 발생 시 선택지 (간헐적)
  └─ 로그/미니맵으로 상황 모니터링

[월 말] 결과 화면
  ├─ 목표 달성 여부
  ├─ 자원 변화, 사건 요약
  └─ 다음 달 설정 화면으로 (재설정)
```

## 🚀 MVP 쪼개기 방향 제안

1. **MVP-01:** 월 1사이클 + 최소 설정(우선순위 1종) + 자동 진행
2. **MVP-02:** 콜로니원 2~3명, 역할/우선순위 세팅
3. **MVP-03:** 자원(식량 등), 생존 실패 조건
4. **MVP-04:** 월간 리포트, 다음 달 설정 연동
5. (이후) 건물, 이벤트, 확장...
